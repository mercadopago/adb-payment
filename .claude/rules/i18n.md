# Internationalisation (i18n)

Translation CSV files for all supported MercadoPago locales.

## Supported Locales

- `en_US.csv` -- English (United States)
- `es_AR.csv` -- Spanish (Argentina)
- `es_CL.csv` -- Spanish (Chile)
- `es_CO.csv` -- Spanish (Colombia)
- `es_MX.csv` -- Spanish (Mexico)
- `es_PE.csv` -- Spanish (Peru)
- `es_UY.csv` -- Spanish (Uruguay)
- `pt_BR.csv` -- Portuguese (Brazil)
- `not_collectable.csv` -- Non-collectable payment strings

## Format

Each CSV line: `"Original string","Translated string"`

## Rules

- Every user-facing string in PHP (`__('string')`) and JS (`$t('string')`) MUST have an entry in ALL locale CSVs
- When adding a new string, add it to ALL 9 CSV files simultaneously
- Keep CSV entries sorted alphabetically for easier diffing
- Do not include HTML in translation strings -- use placeholders (`%1`, `%2`) instead
- Verify translations with native speakers when possible, especially for payment-related terms
- Payment method names should remain consistent across all locales (e.g., "Pix" stays "Pix")
